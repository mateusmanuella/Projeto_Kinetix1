# Sprint 4

## Objetivo da Sprint

Fechar o MVP core com CRUDs principais e integração funcional do frontend.

## Backlog da Sprint

- Finalização da autenticação
- CRUD Paciente
- CRUD Fisioterapeuta
- CRUD Exercício
- Listagem e edição
- Validações completas
- Telas do painel do fisioterapeuta
- Integração frontend com API
- Navegação refinada
- Dashboard simples

## Tarefas dos Desenvolvedores com Notebook

### Backend
- Finalizar segurança
- Implementar CRUD Paciente
- Implementar CRUD Fisioterapeuta
- Implementar CRUD Exercício
- Criar paginação e filtros simples
- Criar validações e mensagens de erro
- Documentar no Swagger

### Frontend
- Implementar login completo
- Criar telas de cadastro e listagem
- Integrar com backend
- Criar dashboard inicial

### Mobile
- Consumir autenticação
- Preparar base de cadastro e login
- Ajustar navegação com backend

### Banco
- Ajustar migrations
- Conferir persistência e integridade

## Critérios de Aceite

- Login operacional
- CRUDs principais funcionando
- Frontend integrado
- Swagger atualizado
- Banco consistente
- Dashboard básico disponível

## Principais Riscos

- CRUDs atrasarem
- Frontend depender de backend incompleto
- Erro de integração

## Mitigação

- Fluxo sequencial
- Teste imediato dos endpoints
- Contratos alinhados

## Entregável para Professores

- Demonstração do MVP Core
- Login funcionando
- CRUDs principais
- Integração frontend-backend
