# Sprint 2

## Objetivo da Sprint

Fechar modelagem, requisitos, rotas da API e protótipos visuais.

## Backlog da Sprint

- Finalização dos requisitos
- Finalização dos casos de uso
- Finalização do DER e MER
- Definição das regras de negócio formais
- Planejamento das tabelas do banco
- Definição das rotas da API
- Descrição de contratos request/response
- Protótipo navegável no Figma
- Organização das dependências entre módulos
- Roadmap técnico das próximas sprints

### Frontend

- Preparar integração futura

### Mobile

### Banco

- Definir constraints
- Preparar script inicial

## Critérios de Aceite

- DER e MER finalizados
- Casos de uso principais concluídos
- Rotas da API documentadas
- Protótipo navegável pronto
- Banco inicial planejado
- Equipe alinhada ao MVP

## Principais Riscos

- Banco complexo demais
- Separação entre documentação e implementação
- Modelo diferente do que pode ser entregue

## Mitigação

- Revisão cruzada
- Validação pelo MVP
- Priorização do que é implementável

## Entregável para Professores

- Modelo de dados fechado
- Casos de uso detalhados
- Rotas da API
- Protótipo maduro
- Plano técnico do MVP
